# AGENTS.md

本文件记录本项目中智能体协作、内容建设和代码实现的约定。所有后续任务都应优先遵守这里的说明。

## 工作方式

- 收到任务后，先用自己的话复述理解：要解决什么问题、交付物是什么。
- 标出不确定但已做出的假设。
- 基于第一性原理思考问题：先解释对象是什么、为什么需要它、它解决什么问题，再展开公式和实现。
- 保持全局视野，大处着眼，小处着手：先确认学习路线和目录位置，再填充具体文件。
- 如果发现用户提出的方案有更好的技术路径，直接说明理由，让用户决定。
- 如果任务反复报错，及时回顾最初方案是否跑偏，不要只在局部堆修补。
- 开发一个较大需求时，所有改动完成后再统一跑一轮测试（不用打开浏览器查看页面效果）；修改过程中不要反复跑测试。
- 每次任务结束前，如果存在 `AGENTS.md`，检查是否需要更新本文件。

## 项目目标

本项目用于系统学习 Linear Algebra（线性代数），目标不是只写静态笔记，而是把知识内容、直观演示、练习和答案放在同一个学习页面里，形成可持续扩展的学习项目。Python 程序只在确实需要数值实验、算法过程或真实应用时添加。

内容应同时覆盖：

- 严谨概念解释。
- 几何直觉。
- 公式推导。
- HTML 交互演示。
- 可选 Python 可视化或数值实验。
- 练习题与思考题。

## 目录组织

知识点采用单页优先方式。一个知识点目录就是一个独立学习单元，默认用 `index.html` 承载知识讲解、互动呈现、练习题和参考答案。

推荐结构：

```text
topics/
  01-vectors-and-linear-combinations/
    index.html
    demo.py
```

文件职责：

- `index.html`：该知识点的主学习页，包含学习目标、概念解释、公式、互动演示、练习题和参考答案要点。
- `demo.py`：可选文件。只有需要算法过程、数值验证、真实数据或 HTML 不适合表达时才创建。

不要把讲解、演示和练习拆到多个文件中，除非单页已经明显过大或存在复用需求。不要把所有 demo 统一丢到远离知识点的全局目录中，除非它们确实是跨章节共享的公共工具。

## 内容风格

- 使用中文讲解，必要时保留英文术语，例如 vector、span、basis、rank、null space、eigenvalue。
- 每个知识点优先回答三个问题：
  - 它是什么？
  - 为什么需要它？
  - 它解决了什么问题？
- 先建立直觉，再给定义和公式，再给例子，最后给练习。
- 避免只堆公式。公式出现时，要说明每个符号的含义和公式背后的几何或计算意义。
- 对抽象概念尽量提供二维或三维可视化解释。
- 对容易混淆的概念，应明确对比，例如 column space 和 null space、basis 和 coordinates、eigenvalue 和 singular value。

## 内容调研流程

- 组织新知识点前，先从第一性原理列出本章核心问题和学习者可能卡住的点，再决定是否调研外部资料。
- 对抽象或关键主题，应做轻量外部调研：至少参考一个严谨来源校准定义，一个直觉型来源借鉴讲法，一个练习或交互型来源参考学习路径。
- 调研目标不是复制资料，而是提炼更清楚的解释顺序、典型例子、常见误区和可操作的交互任务。
- 外部资料只作为教学设计依据，最终页面仍应保持本项目风格：中文讲解、第一性原理、单页学习单元、讲解与交互和练习同页。
- 如果外部来源影响了页面组织，应在知识点页面或任务总结中简要说明采用了哪些教学取舍。

## HTML 演示要求

- 每个知识点的 `index.html` 应能直接用浏览器打开。
- `index.html` 应优先把知识讲解、互动呈现、概念题、手算题、实验题、思考题和参考答案要点放在同一个页面中。
- 优先使用原生 HTML/CSS/JavaScript，除非引入库能明显降低复杂度。
- 交互控件应靠近可视化区域，用户调整参数后应立即看到变化。
- 页面要自包含，尽量不要依赖远程 CDN；如确实需要依赖，应在页面中说明。
- 演示应服务于概念理解，不要做成与知识点无关的炫技页面。
- 对二维线性代数主题，优先使用 canvas 展示坐标轴、网格、向量、变换前后状态。

## Python 演示要求

- Python demo 是可选项，不是每章必需。
- 只有当主题需要数值实验、算法过程、真实数据或批量可视化时才创建 Python 脚本。
- Python demo 优先使用 `numpy` 和 `matplotlib`。
- Python demo 必须通过项目根目录的虚拟环境执行，命令格式为 `.venv/bin/python path/to/demo.py`。
- 项目级 Python 依赖写入 `requirements.txt`，不要只在章节 README 中零散记录。
- 如果需要额外依赖，应在对应知识点 `index.html` 中写清楚安装方式。
- 程序应能独立运行，并在开头用简短注释说明它演示的概念。
- 数值实验应尽量展示“计算结果”和“几何/直观解释”之间的对应关系。
- 不要为了复杂而复杂；教学项目中清晰比高度抽象更重要。

## 练习设计

每个知识点的 `index.html` 至少包含：

- 概念题：检查是否理解定义和直觉。
- 手算题：训练基本计算能力。
- 互动实验题：要求操作当前页面的交互演示。
- 程序题：仅当该知识点存在可选 `demo.py` 时才添加。
- 思考题：连接该知识点和后续主题。
- 参考答案要点：帮助学习者及时校正理解。

练习题应尽量从小例子出发，不要一开始就给过大的矩阵或复杂数据。

3. `topics/03-linear-systems-and-solution-spaces/`

4. PCA 降维。

## 变更约定

- 修改内容时，优先保持根 README、AGENTS.md、知识点 `index.html` 和可选 `demo.py` 之间的一致性。
- 新增知识点时，默认创建 `index.html`。只有需要数值实验、算法过程或真实应用时才添加 `demo.py`。
- 如果只实现其中一部分，应在该知识点 `index.html` 中明确当前状态和待补内容。
- 不要引入与学习目标无关的大型框架。
- 大范围调整目录结构前，先说明迁移理由和影响范围。
