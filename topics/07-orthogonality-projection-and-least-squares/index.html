07. 正交、投影与最小二乘

本页把 dot product、orthogonality、projection 和 least squares 连成一条线：点积衡量方向关系，正交表示没有同向分量，投影给出最近近似，最小二乘就是把不可解的问题投影到可解空间上。

本章要解决的问题

核心问题

当目标不能被精确表示时，怎样找到最接近的可表示结果？

  1. 点积：u·v 衡量两个向量在同一方向上的重合程度。
  2. 正交：u·v=0 表示两个方向互不贡献，是线性代数里的“垂直”。
  3. 投影：把 v 投到一条线或一个子空间上，得到最近的可表示向量。
  4. 残差：v-proj(v) 是没法用目标子空间表达的部分，它和子空间正交。
  5. 最小二乘：当 Ax=b 无解时，改求最接近 b 的 Ax。
  6. 正规方程：最优残差必须垂直于 column space，所以 A^T(Ax-b)=0。

知识讲解

1. 点积是什么

点积把两个向量变成一个数，用来衡量它们方向上的重合程度。

u·v = u1*v1 + u2*v2
u·v = |u||v|cos(theta)

如果夹角小，点积为正；如果夹角大于 90°，点积为负；如果正好垂直，点积为 0。

2. 正交为什么重要

正交意味着“没有沿对方方向的分量”。这比几何里的垂直更有用，因为它可以扩展到高维空间。

u·v = 0

在线性代数中，很多“最优近似”条件最终都会变成某个残差和某个子空间正交。

3. 投影是最近点

把 v 投影到由 u 张成的直线上，得到这条线上离 v 最近的点：

proj_u(v) = (v·u / u·u) u

投影后剩下的残差 v-proj_u(v) 与 u 正交。这就是“最近”的几何原因。

4. 最小二乘解决什么问题

现实中经常出现方程太多、未知量太少的情况，Ax=b 通常无解。least squares 不要求精确等于 b，而是找最接近 b 的 Ax。

minimize ||Ax - b||^2

拟合直线就是典型例子：用两个参数 m,c 尽量解释多组数据点。

5. 正规方程从哪里来

最优近似 Ax_hat 是 b 在 Col(A) 上的投影。残差 r=b-Ax_hat 必须垂直于 Col(A) 的每一列：

A^T(b - Ax_hat) = 0
A^T A x_hat = A^T b

这不是魔法公式，而是“残差与可表示方向正交”的代数写法。

6. 常见混淆

  • 投影不是随便找一个看起来近的点，而是让残差正交。
  • 最小二乘最小化的是残差平方和，不是每个点的误差都一样小。
  • 正规方程可能放大数值误差；真实工程中常用 QR 或 SVD 求解，但本章先建立几何直觉。

本章最重要的一句话：最好的线性近似不是让残差消失，而是让残差垂直于所有可调方向。

概念题

  1. 为什么 u·v=0 可以理解成“v 没有沿 u 的分量”？
  2. 为什么投影点是直线上离原向量最近的点？

手算题

  1. 设 u=[2,0]，v=[3,4]，计算 proj_u(v) 和残差。
  2. 已知三点 (0,1),(1,2),(2,2)，试着说明为什么最佳拟合线的残差和常数列 [1,1,1] 点积应为 0。

思考题

  1. 如果所有数据点的 x 几乎相同，为什么用 y=mx+c 做最小二乘会变得不稳定？

参考答案要点

展开精简答案
  1. 因为 u·v=|u||v|cos(theta)，垂直时 cos(theta)=0，沿 u 的投影长度为 0。
  2. 直线上的任意候选点和投影点相差一个沿直线的向量；残差与这条线正交，所以由勾股关系可知其他候选点距离更远。
  3. proj_u(v)=(v·u/u·u)u=(6/4)[2,0]=[3,0]，残差是 [0,4]。
  4. 常数列是模型矩阵的一列；最优残差必须和模型矩阵每一列正交，所以残差和 [1,1,1] 的点积为 0，也就是残差和为 0。
  5. 当所有 x 几乎相同，斜率方向提供的信息很弱，很多斜率都能给出接近的误差，解对微小扰动很敏感。

教学组织依据

本页采用“点积 -> 正交 -> 投影 -> 最小二乘 -> 正规方程”的顺序，让最小二乘自然成为投影思想的延伸。

  • 练习组织：从正交投影过渡到 least squares，参考 Khan Academy: Orthogonal projections。
