09. 应用专题：推荐系统中的矩阵分解

推荐系统里的矩阵分解把“用户给物品打分”的稀疏矩阵拆成两个低维因子矩阵：用户向量表示偏好，物品向量表示属性，两者点积给出未知评分的预测。

本专题要解决的问题

核心问题

评分矩阵大部分位置都是空的，怎样用少量潜在因子预测用户可能喜欢什么？

  1. 评分矩阵：行是用户，列是物品，已评分位置是观测值，空位置是待预测值。
  2. 低秩假设：用户偏好和物品属性可以由少数潜在因子解释，例如“动作程度”“文艺程度”。
  3. 矩阵分解：用 R ≈ μ + P Q^T，其中 P 是用户因子，Q 是物品因子。
  4. 只拟合观测：目标函数只在已评分位置求误差，缺失值不是 0。
  5. 正则化：惩罚过大的因子，避免为了少量评分把向量拉得过激。
  6. 推荐：对未评分物品计算预测分，按分数排序给出候选推荐。

知识讲解

1. 推荐矩阵是什么

把用户放在行，物品放在列，评分矩阵 R 的元素 R_ui 表示用户 u 对物品 i 的评分。

真实系统中大多数用户只评价过极少数物品，所以 R 是稀疏矩阵。推荐问题的核心是：不能把缺失值当 0，而要预测它们。

2. 低秩假设为什么合理

用户喜欢一部电影，通常不是因为一个孤立编号，而是因为少数潜在因素：类型、节奏、年代、演员、严肃程度等。

如果这些因素的数量远少于用户数和物品数，那么评分矩阵就有低秩结构，可以用少数因子近似。

3. 矩阵分解模型

最基本的模型把评分写成用户因子和物品因子的点积，再加上全局平均分：

R_ui ≈ μ + p_u^T q_i

p_u 是用户 u 的偏好向量，q_i 是物品 i 的属性向量。点积越大，表示用户和物品越匹配。

4. 目标函数只看已评分位置

令 Ω 表示所有已观测评分位置，训练目标是：

min Σ_(u,i in Ω) (R_ui - μ - p_u^T q_i)^2
    + λ(Σ_u ||p_u||^2 + Σ_i ||q_i||^2)

第一项拟合已有评分，第二项限制因子大小。正则化越强，模型越保守。

5. ALS 为什么是最小二乘

如果固定所有物品因子 q_i，每个用户因子 p_u 的求解就是一个小型岭回归；反过来固定用户因子求物品因子也是同样问题。

ALS 交替做这两步。整体目标不是一次性线性问题，但每个子问题都是线性代数中的最小二乘。

6. 和 SVD 有什么不同

普通 SVD 分解完整矩阵；推荐系统只有部分评分。如果先把缺失值填 0 再做 SVD，会把“没看过”误读成“极不喜欢”。

矩阵分解推荐更像矩阵补全：只用观测评分约束模型，再用低秩结构推断空位置。

一句话总结：推荐系统矩阵分解用低维用户向量和物品向量解释已评分位置，再用点积预测缺失评分。

练习

概念题

  1. 为什么推荐系统里的缺失评分不能直接当作 0？
  2. 用户因子和物品因子的点积为什么可以理解成匹配程度？

手算题

  1. 设全局均分 μ=3.5，用户向量 p=[0.8,-0.4]，物品向量 q=[1.0,0.5]。预测评分是多少？
  2. 一个 1000 x 500 评分矩阵如果用 k=20 个因子分解，用户和物品因子共需多少个参数？不考虑偏置时比完整矩阵少多少？

互动实验题

  1. 选择“稀疏冷启动”，切换不同用户，观察评分少的用户推荐是否更容易被全局均值主导。
  2. 把 λ 从 0.01 调到 1.20，观察因子图是否收缩、RMSE 是否变化。
  3. 把 k 从 1 调到 3，观察模型是否更容易拟合噪声评分。

思考题

  1. 为什么观测评分上的 RMSE 越低，不一定代表真实推荐越好？
  2. 如果新用户完全没有评分，只靠矩阵分解能给出个性化推荐吗？还需要什么信息？

参考答案要点

展开精简答案
  1. 缺失表示未知，不表示用户不喜欢；填 0 会把大量未观察位置当作负反馈，严重扭曲模型。
  2. 点积会在相同方向上相互增强：用户偏好强、物品属性也强时预测更高；方向相反时预测更低。
  3. 预测为 3.5 + 0.8*1.0 + (-0.4)*0.5 = 4.1。
  4. 参数量为 20*(1000+500)=30000，完整矩阵有 500000 个数，少 470000 个参数。
  5. 训练 RMSE 低可能来自过拟合已有评分；推荐质量还要看未见数据、排序指标、多样性、新鲜度和业务目标。
  6. 完全没有评分的新用户无法学到个性化用户因子；通常需要热门推荐、内容特征、问卷、上下文或少量初始反馈。

教学组织依据

本页采用“稀疏评分矩阵 -> 低秩潜在因子 -> 只拟合观测位置 -> ALS 最小二乘 -> 推荐排序”的顺序，把推荐系统问题还原成矩阵补全和正则化最小二乘。

  • 算法实现：ALS 交替求解最小二乘的工程表达，参考 Apache Spark MLlib: Collaborative Filtering。
