09. 应用专题：PageRank

PageRank 把网页链接看成一个有向图：一个页面重要，不只是因为很多页面指向它，还因为重要页面把自己的权重分给了它。线性代数上，这就是在一个随机转移矩阵里寻找稳定分布。

核心问题

如果网页之间互相链接，怎样用一个矩阵给每个网页分配“长期被访问的重要性”？

  1. 链接投票：页面 j 指向页面 i，可以理解为 j 把一部分重要性传给 i。
  2. 转移矩阵：把链接图变成列随机矩阵 M，每列表示从一个页面跳到各页面的概率。
  3. 随机游走：排名向量 r 表示一个长期浏览者停在各页面的概率。
  4. 阻尼因子：用 G=αM+(1-α)ee^T/n 避免陷入孤岛或死链。
  5. 稳定分布：PageRank 满足 r=Gr，它是特征值 1 对应的特征向量。
  6. 幂迭代：从均匀向量开始反复算 r_{t+1}=Gr_t，直到排名基本不变。

知识讲解

1. PageRank 是什么

PageRank 是一种给有向图节点排序的方法。网页是节点，超链接是有向边。一个页面的得分来自指向它的页面，但每个来源页面会把自己的得分按出链数平均分出去。

PR(i) = sum over j->i of PR(j) / out_degree(j)

这比简单数入链更合理：来自高质量页面的一条链接，通常比来自无名页面的一条链接更有分量。

2. 为什么需要转移矩阵

把所有链接规则写进矩阵 M。如果页面 j 有 d_j 条出链，并且 j 指向 i，就令：

M[i,j] = 1 / d_j

这样第 j 列表示从页面 j 出发下一步到各页面的概率。排名向量更新就是一次矩阵乘法：r_{t+1}=Mr_t。

3. 死链和蜘蛛陷阱

如果一个页面没有出链，随机浏览者到那里就走不出来，这叫 dangling node。若一组页面只互相链接、不再指向外部，权重也会被困住，这叫 spider trap。

这两个问题都会让朴素随机游走产生不稳定或不合理的排名。

4. 阻尼因子解决什么

PageRank 假设浏览者大多数时候沿链接走，但偶尔会随机跳到任意页面。阻尼后的矩阵是：

G = αM + (1 - α)ee^T / n

α 越接近 1，链接结构影响越强；α 越小，随机跳转越多，排名越接近均匀。

5. 为什么是特征向量

长期排名不是某一步的访问概率，而是反复跳转后不再变化的稳定分布：

r = Gr

这说明 r 是矩阵 G 的特征值 1 对应的特征向量，并且所有分量非负、总和为 1。

6. 幂迭代为什么有效

直接求大型网页图的特征向量很贵。幂迭代只需要不断做矩阵乘向量：从均匀分布出发，反复计算 r_{t+1}=Gr_t。

阻尼让矩阵更容易收敛；在真实规模里，还会利用链接图非常稀疏这一点，不显式存完整矩阵。

一句话总结：PageRank 把“网页重要性”定义成随机浏览者的长期停留概率，计算上就是找一个列随机矩阵的稳定特征向量。

思考题

  1. 为什么 PageRank 能缓解链接数量作弊，但不能完全解决作弊？
  2. PageRank 和“找特征向量”之间的关系，对后续学习马尔可夫链、推荐系统或图神经网络有什么启发？

参考答案要点

展开精简答案
  1. PageRank 会考虑来源页面自身的重要性和出链数；一个高分页面给出的少数链接，比低分页面的大量链接更有权重。
  2. α 小表示更多随机跳转，链接结构被削弱，每个页面都有相近的基础访问概率。
  3. 按页面顺序 A、B、C，矩阵为 [[0,0,1],[1/2,0,0],[1/2,1,0]]。
  4. 基础概率是 (1-0.85)/4=0.0375。
  5. 陷阱网络中，较大的 α 会让权重更依赖链接并更容易困在陷阱；较小的 α 会用随机跳转稀释陷阱。
  6. 新增入链不保证排名上升很多，因为来源页自身分数可能很低，或者它的出链很多，每条链接分到的权重很少。
  7. PageRank 把“重要性”写成一个自洽方程，这种固定点思想也常出现在马尔可夫链稳态、推荐系统迭代更新和图上传播模型里。
